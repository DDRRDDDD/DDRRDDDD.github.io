#### 🚨 직면했던 과제
> **"생소한 Flutter, 하지만 멈출 수 없는 출시 일정"**
> * **기술적 공백과 러닝 커브** : 백엔드 개발자로서 합류했으나, Flutter 경험이 전무한 상태에서 즉시 개발에 투입되어야 하는 기술적 한계가 있었습니다.
> * **생산성 저하 우려** : 학습 곡선으로 인해 전체 프로젝트 속도가 떨어지는 것을 막고, 개발을 겸하는 기획자와의 효율적인 역할 분담이 시급했습니다.

#### 🛠 문제 해결
> **💡 전략적 분업과 기술 주도권 확보**
> * **컴포넌트 우선 개발 전략** : 당장 기여할 수 있는 **UI 위젯** 구현부터 착수했습니다. 개발하며 공부하고, 동료가 비즈니스 로직 연결에 집중할 수 있게 했습니다.
> * **유연한 협업** : '제가 위젯을 조립하고 → 동료가 기능을 붙이는' 협업 방식을 제안하여, 초기 학습 기간에도 개발 속도를 유지하며 생산성을 확보했습니다.
> * **핵심 기능 주도 및 기술 고도화** : 프레임워크 적응 후, 핵심 기능인 **'실시간 채팅'** 구현을 전담했습니다. 기존 코드 베이스에 **Riverpod**를 신규 도입하여, Firebase 데이터 스트림을 구독하고 UI 상태를 갱신하는 로직을 구현했습니다.
