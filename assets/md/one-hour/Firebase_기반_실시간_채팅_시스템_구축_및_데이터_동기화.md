📌 백엔드 개발자가 부재한 상황에서 **Serverless 아키텍처**를 도입하여, 별도의 소켓 서버 없이 실시간 1:1 및 그룹 채팅** 기능을 구현했습니다.

#### 1. 직면했던 과제
> **"서버 리소스 없이 실시간성을 어떻게 확보할 것인가?"**
> * **No Backend**: 소켓 서버(Socket Server)를 구축할 수 없는 환경에서 채팅의 핵심인 '실시간성'을 확보해야 하는 난관이 있었습니다.
> * **Data Sync**: 네트워크가 불안정하거나 앱이 백그라운드에서 돌아올 때도 메시지가 누락되지 않는 견고한 동기화가 필요했습니다.

#### 2. 해결 및 구현 과정
> **💡 Firestore Stream을 활용한 반응형 파이프라인 구축**
> * 전통적인 HTTP Polling 방식 대신, **Firestore의 Snapshot Listener**를 활용했습니다.
> * **`StreamBuilder` 적용:** DB 데이터와 UI를 직접 바인딩하여, 메시지가 생성되는 즉시 화면이 리빌드되는 **Reactive UI**를 구현했습니다.

> **💡 확장성을 고려한 통합 데이터 모델링**
> * **Unified Model**: `ChatRoom` 문서 내에 `participants`(참여자 배열)와 `type`(1:1/Group) 필드를 두어, 하나의 모델로 두 가지 채팅 타입을 모두 처리했습니다.
> * **Ordering**: `serverTimestamp`를 정렬 기준으로 삼아 메시지 순서 꼬임 현상을 원천 차단했습니다.

#### 3. 주요 성과
> 🚀 **Stable Launch**: 소켓 서버 구축 비용 **0원**으로 핵심 기능을 안정적으로 런칭했습니다.
> 🚀 **Zero Data Loss**: 운영 기간 중 메시지 전송 실패나 데이터 유실 사고 **0건**을 기록하며 안정성을 증명했습니다.
