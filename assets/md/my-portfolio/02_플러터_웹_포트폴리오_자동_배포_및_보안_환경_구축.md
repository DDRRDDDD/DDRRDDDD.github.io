#### 🚨 직면했던 과제
> * **수동 배포의 번거로움** : 매번 빌드 명령어를 실행하고 결과물을 직접 업로드해야 해서 개발 흐름이 자주 끊겼습니다.
> * **보안 환경 변수 관리의 복잡성** : 민감한 정보가 담긴 환경 설정 파일을 자동 배포 환경에서 사용하려면, 설정 화면에 일일이 변수를 등록하고 파일로 다시 주입하는 복잡한 과정이 필요했습니다.

#### 🛠 문제 해결 및 구현 과정
> **💡 깃허브 액션을 활용한 배포 자동화 및 하위 저장소(서브모듈) 연동**
> * **배포 전 과정 자동화** : 코드가 반영되면 플러터 및 파이썬 환경 설정부터 웹 빌드, 최종 배포까지 한 번에 처리되도록 작업 흐름을 구성했습니다.
> * **하위 저장소를 활용한 설정 파일 관리** : 민감한 설정 파일을 별도의 비공개 저장소로 분리한 뒤 연결했습니다. 코드 불러오기 단계에서 하위 저장소 옵션을 켜서 설정 파일들을 안전하게 가져오도록 구성했습니다.
> * **안전한 변수 주입** : envied 도구를 활용하여, 가져온 보안 변수들을 빌드 과정에서 코드 내부에 안전하게 난독화하여 적용했습니다.

#### 📈 성과/개선
> * **개발 생산성 향상** : 수동 배포에 들어가던 시간과 노력을 완전히 없애고, 온전히 포트폴리오의 화면과 기능 구현에만 집중할 수 있는 환경을 만들었습니다.
> * **유지보수 편의성 증대** : 번거로운 변수 등록 과정 없이, 개인 개발 환경과 자동 배포 환경 모두 동일한 방식으로 설정 파일을 관리하게 되어 동기화가 매우 간편해졌습니다.
