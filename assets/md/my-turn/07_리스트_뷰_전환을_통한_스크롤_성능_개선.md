#### 🚨 직면했던 과제
> **복잡한 UI 요소로 인한 초기 렌더링 과부하**
> * **과도한 초기 레이아웃 연산**: `Wrap`과 같이 줄바꿈 계산 비용이 높은 위젯을 `Column`에 다수 배치하자, 화면 진입 시 보이지 않는 영역까지 한꺼번에 연산하느라 **프레임이 심하게 떨어지는 현상**이 발생했습니다.
> * **스크롤 버벅임**: 리스트 데이터가 늘어날수록 미리 처리해야 할 레이아웃 부하가 커져, 스크롤을 할 때마다 화면이 매끄럽게 이어지지 못하고 버벅이는 문제가 있었습니다.

#### 🛠 문제 해결 및 구현 과정
> **💡 ListView 도입으로 연산 시점 분산**
> * **렌더링 방식 변경**: 전체 높이를 미리 확정해야 하는 `SingleChildScrollView`+`Column` 대신, 화면에 보이는 영역만 채우는 방식인 `ListView`로 구조를 변경했습니다.
> * **지연된 레이아웃**: `ListView` 내부적으로 화면 밖 요소들의 레이아웃 연산을 수행하지 않는 특성을 활용하여, `Wrap` 위젯의 무거운 연산이 스크롤 시점에 나누어 처리되도록 만들었습니다.

#### 📈 성과/개선
> * **진입 성능 개선** : 불필요한 초기 연산을 제거하여, 데이터가 많은 목록 화면에 진입할 때 발생하던 프레임 드랍 현상을 해결했습니다.
> * **스크롤 안정성 확보** : 레이아웃 연산 부하가 적절히 분산됨에 따라, 복잡한 UI 구성에서도 끊김 없이 부드러운 스크롤 경험을 제공했습니다.
