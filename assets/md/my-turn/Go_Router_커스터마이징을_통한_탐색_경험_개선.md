#### 🚨 직면했던 과제
> **사용자의 탐색 맥락은 끊기지 않고 유지되어야 한다"**
> **GoRouter 리다이렉트의 구조적 한계** : GoRouter의 기본 `redirect` 로직은 스택을 교체하는 `go` 방식만 지원하여, 로그인 페이지 진입 시 뒤로 가기가 불가능해지는 등 **이전 화면의 맥락을 보존해야 하는 시나리오에 대응하기 어려운 제약**이 있었습니다.
> **멀티 네비게이션의 상태 초기화** : 바텀 탭과 상단 탭이 중첩된 복합 네비게이션 구조에서, **메인 탭을 전환할 때마다 서브 탭의 탐색 히스토리와 스크롤 위치가 초기화**되어 사용자 경험이 끊기는 문제가 발생했습니다.

#### 🛠 문제 해결 및 구현 과정
> **💡 Push를 지원하는 커스텀 가드 구현**
> * **라우팅 미들웨어 확장**: 프레임워크가 제공하는 기본 리다이렉트 대신, 진입 조건에 따라 `push` 명령을 수행할 수 있는 **커스텀 가드 로직을 미들웨어 형태로 구현**했습니다.
> * **유연한 분기 처리**: 이를 통해 특정 조건(예: 비로그인) 발생 시 현재 스택을 유지한 채 새로운 화면을 쌓을 수 있게 되어, **상황에 맞는 유연한 라우팅 분기 처리**가 가능해졌습니다.
>
> **💡 중첩된 네비게이션의 상태 보존**
> * **StatefulShellRoute 활용**: 탭마다 독립적인 네비게이터 키를 부여하고, 위젯 트리 자체를 메모리에 유지하는 `StatefulShellRoute`를 적용했습니다.
> * **탐색 맥락 유지**: 탭 간 이동 시에도 각 탭의 네비게이션 스택과 스크롤 상태를 그대로 보존하여, 사용자가 **어느 시점에 돌아와도 작업을 이어서 할 수 있는 환경**을 구축했습니다.

#### 📈 성과/개선
> * **유연한 화면 전환 로직 확보** : `go`와 `push`를 상황에 맞춰 선택적으로 사용할 수 있게 되어, 로그인 플로우 등에서 **자연스러운 뒤로 가기 경험(UX)**을 제공했습니다.
> * **네이티브 수준의 탐색 경험 구현** : 복잡한 탭 구조에서도 사용자의 탐색 경로와 데이터가 휘발되지 않고 유지되어, **앱의 사용성이 대폭 향상**되었습니다.
