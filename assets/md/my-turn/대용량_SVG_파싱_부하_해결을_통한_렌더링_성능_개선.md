#### ❝보드게임 특성상 화면당 50개 이상의 복잡한 벡터 아이콘을 렌더링해야 했으나, 기존 런타임 XML 파싱 방식은 심각한 프레임 드랍을 유발했습니다.❞

#### 🚨 문제(Problem)
> * 초기 진입 시 메인 스레드가 SVG XML 파싱 연산에 잠식되어 UI가 멈추는 현상 발생.
> * 리스트 스크롤 시마다 파싱이 반복되어 60fps 방어가 불가능함.

#### 🛠 해결(Solution)
> * **사전 컴파일(AOT) 파이프라인 구축** : vector_graphics_compiler를 도입하여 빌드 시점에 SVG를 최적화된 바이너리 포맷(.vec)으로 변환.
> * **런타임 파싱 제거** : 앱 실행 시에는 별도의 연산 없이 바이너리 데이터를 GPU 텍스처로 즉시 업로드하는 구조로 변경.

#### 📈 성과(Outcome)
> * 이미지 렌더링 비용 **90% 절감** (DevTools Profiler 기준).
> * 저사양 안드로이드 기기 및 웹 환경에서 스크롤 끊김 현상(Jank) 완전 제거.
