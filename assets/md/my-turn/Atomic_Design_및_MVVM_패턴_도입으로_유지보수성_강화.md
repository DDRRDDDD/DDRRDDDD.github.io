#### ❝코드는 더 엄격하게 관리되어야 합니다. 확장 가능한 컴포넌트 시스템을 구축했습니다.❞

#### 🚨 문제(Problem)
> * UI와 비즈니스 로직의 결합도가 높아, 디자인 변경 시 로직까지 수정해야 하는 유지보수 위험 존재.
> * 비슷한 UI 요소가 여러 곳에서 중복 구현되는 비효율.
> * 디자이너와 개발자 간 협업 시 컴포넌트 명세 불일치로 인한 재작업 발생.

#### 🛠 해결(Solution)
> * **Atomic Design 시스템** : 컴포넌트를 Atoms -> Molecules -> Organisms 단위로 분리하여 조립 가능한 UI 라이브러리 구축.
> * **MVVM 패턴** : Riverpod을 ViewModel로 활용하여 View는 철저히 상태를 그리는 역할에만 집중하도록 관심사 분리(SoC).

#### 📈 성과(Outcome)
> * UI 컴포넌트 재사용률 극대화로 개발 속도 향상.
> * 디자인 변경 요구사항에 유연하게 대처 가능한 코드 베이스 확보.
