#### ❝Riverpod 선행 로딩으로 데이터 재요청 오버헤드 제거❞

#### 🚨 문제(Problem)
> * 목록과 상세 페이지를 오갈 때마다 동일한 데이터를 서버에 재요청하여 로딩 딜레이 발생.

#### 🛠 해결(Solution)
> * **상태 보존** : `ref.keepAlive()`로 데이터 수명을 연장하고, 화면 진입 전에 필요한 데이터를 백그라운드로 미리 로드하는 Prefetching 패턴 적용.
> * **선행 로딩** : 사용자 행동 예측을 통해 필요한 데이터를 백그라운드에서 미리 로드.

#### 📈 성과(Outcome)
> * 페이지 전환 시 즉시 로딩 경험(Zero-Latency) 제공.
> * 사용 중 불필요한 네트워크 트래픽 제거.
