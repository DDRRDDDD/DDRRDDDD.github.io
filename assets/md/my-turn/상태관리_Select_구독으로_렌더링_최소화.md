#### ❝상태 변화가 UI 리빌드로 이어지는 범위를 외과수술하듯 정밀하게 제어하여 퍼포먼스를 확보했습니다.❞

#### 🚨 문제(Problem)
> * 거대한 상태 객체를 구독하면 객체 내 관련 없는 필드 변경에도 위젯이 재빌드되어 성능 저하가 발생함.

#### 🛠 해결(Solution)
> * **Selector 패턴 적용** : `ref.watch(provider.select(...))`를 도입해 위젯이 상태의 특정 필드 변화만 감지하도록 제한.
> * **불변 객체 비교** : 불변성 원칙과 원시 타입 비교(int/String)를 결합해 실제 값 변화가 없으면 리빌드를 차단.

#### 📈 성과(Outcome)
> * 상태 변경 시 리빌드 대상이 전체 트리의 **1/N** 수준으로 축소.
> * 복잡한 화면에서도 높은 UI 응답성(120fps 근접)을 확보.
